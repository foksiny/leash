Installation & Setup

Before using Leash, you need to install its dependencies: Python 3 for running the compiler, LLVM development libraries for code generation, and a C compiler (GCC or Clang) for linking the runtime garbage collector. Leash itself is then installed via pip.

Prerequisites

Python 3Running the Leash compiler3.8+
LLVMCode generation via llvmlite11+ (development libraries)

# Install LLVM development libraries and a C compiler
sudo apt install llvm-dev gcc

# Optional: cross-compilation to Windows (MinGW)
sudo apt install gcc-mingw-w64-x86-64

sudo dnf install llvm-devel gcc

Install Leash

Once the prerequisites are installed, install Leash and its Python dependencies. The -e flag installs in editable mode, which is useful during development as changes to the source are immediately reflected.

# Navigate to the Leash project directory, then:
pip install -e .

This installs two CLI tools:

  • leash — the Leash compiler CLI for compiling, running, and analyzing programs
  • leashed — the Leash package manager for publishing and installing libraries
# Verify installation
leash --version
leashed --version

Running Leash Programs

You can run Leash files (.lsh) directly using the run command, or compile them to a native executable using the compile command. Leash also supports project-level workflows with init, build, and runp for larger projects with configuration files.

Direct Execution

To execute a Leash program without creating an output binary executable (and optionally pass command line arguments):

python3 -m leash.cli run program.lsh [arguments...]

Compiling to a Binary

# Default output (same name as input)
python3 -m leash.cli compile program.lsh
./program

# Custom output name
python3 -m leash.cli compile program.lsh to my_custom_binary
./my_custom_binary

# Dynamic library (.so)
python3 -m leash.cli compile program.lsh to-dynamic mylib

# Static library (.a)
python3 -m leash.cli compile program.lsh to-static mylib

# Link with external libraries
python3 -m leash.cli compile program.lsh -lm
python3 -m leash.cli run program.lsh -lcurl

Dumping LLVM IR

python3 -m leash.cli dump program.lsh
python3 -m leash.cli dump program.lsh to myoutput

Project Workflow

Project Init (init)

The init command scaffolds a new Leash project with the standard directory structure, including a main entry point, import and output directories, and a project configuration file. This provides a consistent starting point for any new Leash project.

leash init my_project

Creates the standard project structure:

my_project/
├─ src/
│   └─ main.lsh          # Entry point
├─ imports/               # Local import directory
├─ out/                   # Compiled output directory
└─ config.lshc            # Project configuration

Generated main.lsh:

fnc main |> show("Hello, World!");

Generated config.lshc:

main: "src/main.lsh"
clibs: {}
imports: "imports/"
opt_level: "O3"

Project Build (build)

The build command reads config.lshc from the current directory and compiles the project. This uses the project configuration to find the main entry file, resolve imports, link C libraries, and apply the configured optimization level.

leash build
leash build --other-imports path/to/imports
leash build -oi path/to/imports

Project Run (runp)

The runp command compiles and runs a project in one step, using config.lshc. The output binary is temporary and cleaned up after execution, making it ideal for development and testing.

# Compile and run
leash runp

# Pass arguments to the program (after --)
leash runp -- arg1 arg2 arg3

# With additional import directories
leash runp --other-imports path/to/imports -- arg1 arg2

Updating Leash

The update command checks for new releases on the Leash GitHub repository and pulls the latest changes. It fetches the latest release tag from github.com/foksiny/leash via the GitHub API, then runs git pull to update your local clone to the latest commit. Requires Git to be installed and the Leash repo to have been cloned from GitHub.

leash update

Additional Import Paths

The --other-imports (or -oi) flag adds extra directories to the module search path for use ...; statements. This is available on compile, run, dump, build, and runp. These directories are searched before ~/.leash/libs/, making it useful when working outside a project setup or when you need to import from non-standard locations.

leash compile program.lsh --other-imports mylibs/
leash run program.lsh -oi mylibs/ -oi other_libs/
