Leash

A strongly-typed, modern compiled programming language built on LLVM.

Leash combines an intuitive syntax with native performance, targeting developers who want the control of a systems language without the ceremony. It compiles directly to machine code via LLVM, features a built-in garbage collector, and ships with a package manager, standard library, and cross-platform support.

Key Features

Strongly Typed

Full type inference with explicit bit-width integers, floats, generics, and tagged unions.

LLVM-Powered

Compiles to native code via LLVM with multiple optimization levels including LTO and PGO.

Zero-Cost Abstractions

Classes, generics, lambdas, and operator overloading compile down to efficient machine code.

Built-in GC

Automatic memory management with an optional manual mode (nogc) for low-level control.

Package Manager

leashed lets you publish, install, and manage libraries with semantic versioning.

Cross-Platform

Compile for Linux, Windows, and macOS from a single source with conditional compilation.

Concurrency

Built-in worker threads with shared and fusion variables for safe multi-threaded programming.

Rich Standard Library

Vectors, matrices, hash tables, file I/O, string utilities, math functions, and windowing.

Quick Start

fnc main() : void {
    show("Hello, World!");
}
# Install
pip install -e .
leash --version

# Run directly
python3 -m leash.cli run hello.lsh

# Compile to native binary
python3 -m leash.cli compile hello.lsh
./hello

Compiler Pipeline

Leash transforms source code into a native executable through a multi-stage pipeline. Each stage feeds its output into the next, from raw source text all the way to a linked binary. The pipeline includes lexical analysis, parsing, type checking, optimization at both the AST and LLVM IR levels, code generation, and final linking with the garbage collector runtime.

Here are the major stages:

.lsh source
  ├─ Lexer (characters → tokens)
  ├─ Parser (tokens → AST)
  ├─ Import Resolution
  ├─ Conditional Compilation
  ├─ Macro Expansion
  ├─ Type Checker
  ├─ Low-Level Checker
  ├─ AST Optimization
  ├─ Code Generation (AST → LLVM IR)
  ├─ Alloca Hoisting
  ├─ Re-parse & Verify
  ├─ LLVM Optimization
  ├─ Object Emission
  ├─ C Runtime Compilation
  └─ Linking → Native Binary
