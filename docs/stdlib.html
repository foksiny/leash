Standard Library Reference

Leash ships with a set of standard libraries in the installthis/ directory. These libraries provide common utilities like type aliases for explicit bit-width types, math functions, string manipulation, tuple containers, and a windowing API. Install them globally to make them available from any project via the use statement.

The standard libraries are optional — you can install only the ones you need, or write your own libraries following the same patterns.

Installation

# Install all standard libraries globally
leash install installthis/

# Then import in any project
use types::*;
use math::Math;
use str::Str;

Library Imports

Basic Import Syntax

use module_name::ItemName;
use hash::Hash;
use colors::Color;

// Import all public items
use mylib::*;

Private Imports

// Import without re-exporting
priv use mylib::*;

pub fnc helper() : int {
    return lib_priv();  // can use private items
}

Nested Paths

use math::operations::add;
use utils::string_helpers::capitalize;

The compiler searches for modules in two locations:

  1. The local directory of the file containing the use statement (including subdirectories)
  2. The global libraries directory (~/.leash/libs/) — where installed libraries are available from anywhere

When you import a module, all its public definitions (structs, classes, enums, functions, templates, etc.) become available in your file. This makes it easy to organize code into reusable, namespaced modules.

Tuple Library (tuple.lsh)

A generic Tuple<T> class for storing a collection of values. Tuples provide a lightweight way to group items without defining a custom struct, with methods for indexed access, membership testing, and size queries.

MethodDescription
Tuple.new(vals T[])Create a new Tuple from an array
.get(idx int)Get the element at index idx
.isin(val T)Check if a value exists
.sizeNumber of elements (property)
use tuple::Tuple;

fnc main() {
    t: Tuple<int> = Tuple.new({1, 2, 3});
    show(t.get(1));   // 2
    show(t.isin(3));  // true
}

Hot Reloader (hotreload.lsh)

The Reloader watches a Leash source file for changes and automatically re-runs it. This is particularly useful during development when you want a live-reload workflow — edit your code and see the results immediately without manually re-running the compiler.

use hotreload::Reloader;

fnc main() {
    r: Reloader = Reloader.new("main.lsh");
    r.start();
}

Window Library (utils::window)

The low-level lshraylib binding is unstable and buggy. The utils::window library is the new, in-development replacement that wraps raylib in a Leash-native API.
use utils::window::*;

wind: LshWindow;
pos: Vector2;
speed: int = 5;

fnc start() {
    pos = Vector2 { x: wind.width / 2, y: wind.height / 2 };
}

fnc update() {
    sizes := Vector2 { x: 50, y: 50 };
    Draw.draw_rect(pos, sizes, WHITE);

    if Input.key_down(KEY_W) { pos.y -= speed; }
    also Input.key_down(KEY_S) { pos.y += speed; }
    also Input.key_down(KEY_A) { pos.x -= speed; }
    also Input.key_down(KEY_D) { pos.x += speed; }
}

fnc main {
    wind = create_window(800, 600, "My Game");
    close_key(KEY_ESCAPE);
    init_window(wind, &start, &update, &endd);
}
