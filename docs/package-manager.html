Leashed Package Manager

leashed is the official package manager for Leash. It lets you create, publish, share, and install Leash libraries through a central registry at github.com/foksiny/leash-packages. Every library lives in its own GitHub repository; the registry is an index.json that maps library names to repo URLs.

Installation

# Installed alongside the Leash compiler
pip install -e .
leashed --version
# leashed v0.1.0

Creating a Package

Scaffold a new library project with leashed init. This creates the standard library structure with a package configuration file where you specify the library's name, version, author, and entry point.

leashed init mylib
cd mylib

Creates:

mylib/
├─ src/
│   └─ main.lsh        # Library entry point
├─ leash-pkg.lshc     # Package configuration
└─ .gitignore

Package Configuration (leash-pkg.lshc)

name: "mylib"
version: "0.1.0"
author: "your-github-username"
description: "The mylib library"
main: "src/main.lsh"

To link to an existing GitHub repo, add a repo field:

repo: "https://github.com/you/mylib.git"

Publishing a Package

From inside your project directory, publish your library to the central registry. The publish command handles everything from verification to registration, making it easy to share your libraries with the community.

leashed publish

This does the following automatically:

  1. Verifies your source code with leash check
  2. Compiles your library to a static .lib / .a file
  3. Creates a GitHub repo under your account (if no repo is set) via gh repo create
  4. Pushes the source and compiled library to your repo
  5. Registers the library in the central registry:
    • If you're the registry owner — pushes directly to index.json
    • If you're a contributor — forks the registry, updates index, and creates a pull request
Requires GitHub CLI (gh) installed and authenticated (gh auth login), and Git installed.

Installing Libraries

Install a library globally so any Leash program can use it. The installer fetches the registry, finds the library's repository, clones it, and places the files in the global libraries directory — no manual configuration needed.

leashed install mylib

This process:

  1. Fetches the registry index.json from raw.githubusercontent.com (no clone needed for the registry itself)
  2. Looks up the library name and finds its repo URL
  3. Shallow-clones only the library's repository (efficient, no history)
  4. Copies the files to ~/.leash/libs/<name>/
  5. Creates a stub <name>.lsh so use mylib::*; resolves correctly
use mylib::*;

fnc main() {
    show(greet("World"));
}

Adding Dependencies to a Project

Add a library as a dependency of the current project. This is the standard way to declare and manage project dependencies — it installs the library, records the dependency in your package config, and sets up the import automatically.

leashed add mylib

This process:

  1. Installs the library globally if not already installed
  2. Appends mylib@version to the dependencies field in leash-pkg.lshc
  3. Inserts use mylib::*; into your main source file

Searching for Libraries

Search the registry for libraries by name or description. The search is efficient because it only fetches the lightweight index.json file — no cloning of individual library repositories is needed until you decide to install one.

leashed search math

How Publishing Works Under the Hood

Each library has its own GitHub repository. The central registry (foksiny/leash-packages) is an index.json file:

{
  "libraries": {
    "mylib": {
      "repo": "https://github.com/you/mylib.git",
      "description": "The mylib library",
      "author": "you",
      "version": "0.1.0"
    }
  }
}

Security Checks

CheckDescription
Package name validationOnly [a-zA-Z_][a-zA-Z0-9_-]* names allowed
Source verificationCode must pass leash check
Publisher identityOnly the original author can update an existing entry
Compilation checkLibrary must compile to a static library

Global Library Installation

Leash also provides a lower-level install command on the leash CLI to copy library files into ~/.leash/libs/ without the package manager.

Installing a Single File

leash install path/to/mylib.lsh

Installing a Directory

leash install path/to/mylib_folder/

The contents of the directory are copied directly into ~/.leash/libs/. The top-level folder name is not preserved.

For example, given:

mylib_folder/
├─ hash.lsh
└─ utils/
    └─ string_helpers.lsh

After installation, ~/.leash/libs/ contains:

~/.leash/libs/
├─ hash.lsh
└─ utils/
    └─ string_helpers.lsh
If a file or directory with the same name already exists in the global libs directory, the installation will fail. Remove the existing library first to replace it.
